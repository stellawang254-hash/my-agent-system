# Skill: Book Deconstruction

## Description
把一本书、一篇长文、一组阅读材料，从“读过”整理成“可复用的知识结构”。

这个 skill 不是只做摘要，也不是简单摘抄金句。
它的核心任务是：

1. 提炼核心主题
2. 识别关键观点
3. 拆出重要概念、方法、案例
4. 整理作者的逻辑链
5. 形成可复习、可写作、可分享、可继续扩展的拆书成果

适用于：
- 商业书
- 方法论书籍
- 认知成长类书籍
- 人物思想类文本
- 股东信
- 演讲稿
- 长文章
- 多篇同主题材料

## When to use
- 帮我拆这本书
- 不要只做摘要
- 帮我提炼这本书的核心观点
- 帮我梳理结构
- 帮我分析作者逻辑
- 帮我整理读书笔记
- 帮我做知识框架
- 帮我做成后续可写作的素材
- 帮我把这组材料拆成知识库

## Workflow

### Step 1：识别任务边界
确认材料完整性、用户最终用途、是否为阶段性拆解。

### Step 2：识别书 / 材料类型
- 观点型
- 方法型
- 认知型
- 案例型
- 人物型
- 体系型

### Step 3：提炼核心问题
回答：
- 作者为什么写这本材料
- 真正想解决什么问题
- 想纠正什么误解
- 最想让读者明白什么

### Step 4：搭整体结构
整理：
- 核心主题
- 全书主线
- 章节结构
- 内容分层
- 重复出现的重点

### Step 5：提炼关键观点
说明：
- 观点是什么
- 为什么这么判断
- 为什么重要
- 能用在哪些场景

### Step 6：提炼关键概念
说明：
- 概念名
- 作者定义
- 通俗解释
- 为什么重要
- 常见误解
- 现实用法

### Step 7：提炼方法或模型
说明：
- 方法名
- 解决问题
- 核心步骤
- 适用场景
- 常见误区
- 简单例子

### Step 8：提炼案例
说明：
- 案例是什么
- 为什么讲它
- 说明了什么
- 对应哪个观点或方法

### Step 9：识别作者逻辑链
尽量回答：
- 作者前提是什么
- 关键判断是什么
- 用了哪些证据
- 怎么推到结论
- 依赖哪些假设

### Step 10：识别易错点
说明误读风险和适用边界。

### Step 11：做现实迁移
补充：
- 适合谁
- 适合解决什么问题
- 对现实有什么启发
- 可以先做哪 3 件事

## Default Output Modes
### Mode A：标准拆书版
- 这本书到底在讲什么
- 作者真正想解决什么问题
- 全书结构梳理
- 最重要的 5 个观点
- 最重要的 3 个概念
- 最值得复用的 3 个方法
- 最有代表性的案例
- 作者的底层逻辑
- 最容易被误解的地方
- 对现实有什么启发
- 一句话总结

### Mode B：知识卡片版
- 核心主题卡
- 观点卡
- 概念卡
- 方法卡
- 案例卡
- 金句卡
- 作者逻辑卡
- 应用场景卡

### Mode C：内容准备版
- 可写作观点清单
- 可展开的小标题
- 可引用的核心表达
- 可做课程的框架
- 可做 PPT 的结构

## One-line Positioning
拆书，不是把内容变短。
而是把一本书整理成以后还能反复调用的知识结构。

# 路由规则说明

## 一、任务优先级判断
默认优先级如下：

1. 先判断是不是分析任务
2. 再判断是不是人物思想任务
3. 再判断是不是输出任务
4. 最后判断是不是知识库沉淀任务

默认顺序：

分析层 → 输出层 → 沉淀层

## 二、典型路由场景

### 场景 1：用户给一本到多本书
默认：
先 book-deconstruction

### 场景 2：用户给同一人物多份材料
默认：
先 author-thinking-analysis

### 场景 3：用户明确说“改写”“润色”“公众号”
如果分析够用：
直接 article-rewriting
否则：
先回到分析层

### 场景 4：用户明确说“知识库”“Obsidian”“卡片”
如果内容结构已成型：
knowledge-base-formatting
否则：
先做分析

## 三、目标导向规则

### 最终目标是“看懂”
优先：
- book-deconstruction
- author-thinking-analysis

### 最终目标是“发文”
优先：
- 先分析
- 再 article-rewriting

### 最终目标是“长期沉淀”
优先：
- 先分析
- 再 knowledge-base-formatting

### 最终目标是“讲课 / PPT”
优先：
- 先分析
- 再输出结构化内容准备版
- 必要时再改写讲稿

## 四、一句话原则
流程乱的时候，不要先问“怎么写”，
先问“现在这一步到底该拆、该提炼，还是该改写”。

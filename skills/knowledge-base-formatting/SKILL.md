# Skill: Knowledge Base Formatting

## Description
把已经分析过、拆解过、整理过的内容，
进一步转成适合知识库长期沉淀的结构化 Markdown 资产。

它不负责第一次分析，
而负责把已有内容整理成：

1. 索引页
2. 卡片页
3. 双链结构
4. 标签结构
5. 人物 / 概念 / 方法 / 案例等知识节点
6. 后续可持续扩展的知识库格式

## When to use
- 帮我整理成知识库格式
- 帮我拆成卡片
- 帮我做索引页
- 帮我生成适合 Obsidian 的结构
- 帮我做双链结构
- 帮我整理标签
- 帮我整理成长期可用的知识节点
- 帮我整理成 vault 结构

## Workflow

### Step 1：识别知识对象
判断更适合拆成：
- 人物
- 概念
- 方法
- 案例
- 主题
- 项目
- 索引页

### Step 2：确定层级结构
- 第一层：索引层
- 第二层：主题层
- 第三层：节点层

### Step 3：拆节点
常见节点：
- 人物节点
- 概念节点
- 方法节点
- 案例节点
- 主题节点

### Step 4：建立双链
优先建立：
- 人物 → 观点 / 概念 / 方法
- 概念 → 人物 / 书籍 / 方法
- 方法 → 概念 / 案例
- 案例 → 方法 / 主题
- 主题 → 全部核心节点

### Step 5：整理标签
只保留高频、稳定、可检索的标签。

### Step 6：输出索引页
索引页负责导航，不负责展开全部内容。

### Step 7：补元数据
必要时加入简洁 YAML。

## Default Output Modes
### Mode A：基础知识库版
- 文件夹建议
- 索引页
- 3-10 个核心节点示例
- 标签建议

### Mode B：Obsidian 版
- YAML
- 双链写法
- 索引页
- 节点页
- 相关链接字段

### Mode C：Agent 友好版
- 节点清楚
- 文件命名稳定
- 目录层级简单
- 每页不过长

## One-line Positioning
不是把内容存起来，
而是把内容整理成以后能继续长、继续连、继续调用的知识结构。
